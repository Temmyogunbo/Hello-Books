    Introduction

    Welcome to HellBooks API! You can use our API to access HelloBooks API endpoints, which can get information on various books, categories, users, and users history in our database.

    You can view code examples in the dark area to the right.

    HTTP Request url must prefix the base url

    BASE URL = http: //emmanuelhellobooks.herokuapp.com/api/vi

    Authentication

    The application uses Json web token simply known as JWT to authenticate user. The JWT token must be signed by the application and it expires after 24 hours. Also, the token must be included in every request. It should be in form of a bearer and token

    authenticate-token:Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...

    Users

    Create a user

    This endpoint create a user record. If record was sucessfully created, user gets a token that will be used for subsequent request.

    HTTP Request

    POST /users/signup

    HTTP Respone

    201 created

    Request body

    {
  "fullName": "Emmanuel Ogunbo",
  "userName": "emmy",
  "email": "johndoe@gmail.com",
  "password": "password",
  "confirmPassword": "password"
}

    Response body

    {
  "fullName": "Emmanuel Ogunbo",
  "userName": "emmy",
  "email": "johndoe@gmail.com",
  "role": "users",
  "membership": "gold",
  "success": true,
  "msg": "Registration successful",
  "id": 1,
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..." 
}

    Signin user

    This endpoint signs in a user into the application. The user gets a token upon successful sign in and it can be used for subsequent request.

    HTTP Request

    HTTP Response

    "userName": "emmy",

  "success": true,

    Change Password

    This endpoint allows user to change their existing password after signing in.

    HTTP Request

    POST /users/change-password

    HTTP Response

    204

    Request body

    {
  "oldPassword": "oldPAssword",
  "newPassword": "newPasword",
  "userName": "emmy" 
}

    Errors

    The HelloBooks API uses the following error codes:

    Error Code Meaning
    400 Bad Request -- Validation errors.
    401 Unauthorized -- Your supplied a wrong API key.
    403 Forbidden -- The user is not allowed to access the resource.
    404 Not Found -- The specified resource could not be found.

    500 | Internal Server Error -- We had a problem with our server. Try again later.
